## Purpose

This module defines everything the system **shows and produces** for humans. It is the final layer of every agent cycle. Raw agent decisions are translated into readable reports, a role-gated dashboard, and downloadable PDF documents that can be used as formal project records.

## Dashboard Specification (Streamlit)

### Navigation & Role Gating

```
Login → Role detected → Routed to role-specific view

Roles:
  contract_manager  → Modules: Contract Upload, Rule Store Review, Project Setup
  site_engineer     → Modules: MPR Upload, Hindrance Register, QA Entry
  project_manager   → Modules: All Projects Overview, Risk Dashboard, EoT Approvals
  contractor        → Modules: My Compliance Status, Notices Received, FM/EoT Claims
  auditor           → Modules: Full Read-Only View, Report Download, Audit Trail
```

### Pages

**Page 1: Project Overview (all roles)**
- Project card: name, contract type, value, day number / SCD
- Progress ring: planned vs actual physical %
- S-curve chart (Plotly): planned vs actual vs forecast
- Risk score badge: colour-coded LOW / MEDIUM / HIGH / CRITICAL
- Active violations count

**Page 2: Compliance Status (contract_manager, project_manager, auditor)**
- List of active compliance events (filterable by severity)
- Each event: title, clause, severity badge, financial impact, action button
- Penalty ledger table: running LD total, cap utilisation progress bar
- EoT decisions log

**Page 3: MPR Upload (site_engineer)**
- Upload `.md` file
- Validation feedback (real-time): ✅ / ❌ per field
- Preview of parsed data before submission
- Hindrance Register entry form (inline)

**Page 4: Risk & Predictions (project_manager, auditor)**
- Risk score gauge
- SHAP waterfall chart (top factors)
- Forecast table (next milestone)
- External signals panel (weather, news)
- Recommendations checklist

**Page 5: Notices & Claims (contractor)**
- Received notices (NCRs, LD notices, Show Cause, Notice of Intent)
- FM claim submission form
- EoT application form
- Variation order claim form
- Dispute initiation button

**Page 6: Audit Trail (auditor)**
- Full event log with agent attribution
- Agent invocation log
- Download: `compliance_{project_id}.pdf`
- Download: `predictions_{project_id}.pdf`
- Download: `penalty_ledger_{project_id}.pdf`

## PDF Export

Generated using `fpdf2`. One click from dashboard.

**compliance.pdf** contains:
- Cover page: project name, authority logo, report date
- Executive summary table
- All active events (with clause references, calculations shown)
- Penalty ledger
- Contractor rights section
- Signature block (Engineer-in-Charge / Authority's Engineer)
- Full audit trail

**predictions.pdf** contains:
- Risk score summary with gauge graphic
- S-curve chart
- Top risk factors (SHAP chart)
- Milestone forecast table
- External signals
- Recommendations

Both PDFs are **numbered, dated, and version-stamped** for use as formal project records in case of dispute or audit.

---

## Report Consumed by Agents

The `compliance.md` and `predictions.md` are also **machine-readable inputs** for the next agent cycle. The Orchestrator reads them to check:
- Which events are still open (and whether cure periods have expired)
- Whether the risk score has changed since the last cycle
- Whether any EoT or refund was decided but not yet actioned
- Whether escalation timers have been triggered

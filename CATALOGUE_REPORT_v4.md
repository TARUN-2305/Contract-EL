### ⚠️ EOT BUG 1 — HR-002 returns 21 days approved instead of 13 days (MEDIUM — overlap deduction not applied at individual hindrance level)

**Expected:** HR-002 (21 days, Oct 4–25) overlaps HR-003 (Oct 12–24) by 12 days → net = 21-12 = 9 days for HR-002 alone, but combined with HR-001 the gross/net calculation should give 13 days approved for HR-002.

**Root cause in `agents/eot_agent.py` `process_hindrance_eot()`:**

```python
# Current code (line ~160):
net_days, overlap = calculate_net_eot(hindrances, today=today)
approved = min(net_days, claimed_days) if claimed_days > 0 else net_days
```

`calculate_net_eot(hindrances)` runs on ALL hindrances (all 6), producing the total net (95 days with `today=Jan 7 2026`). It then compares this total against the single hindrance's `claimed_days` (21). `min(95, 21) = 21` → approves 21. 

The overlap deduction is computed globally but the `approved` is set to `min(global_net, single_claimed)` — this loses the per-hindrance overlap logic entirely.

**Fix in `agents/eot_agent.py`, `process_hindrance_eot()`:**
```python
# WRONG (current):
net_days, overlap = calculate_net_eot(hindrances, today=today)
approved = min(net_days, claimed_days) if claimed_days > 0 else net_days

# CORRECT: Calculate overlap only for this hindrance against others
# Find the target hindrance's date range
target_start = _parse_date(hindrance.get("date_of_occurrence"))
target_end = _parse_date(hindrance.get("date_of_removal")) or today

# Calculate gross days for this hindrance
gross_this = (target_end - target_start).days

# Calculate overlap with OTHER hindrances (not this one)
other_hindrances = [h for h in hindrances if h.get("hindrance_id") != hindrance_id]
overlap_days = 0
for other in other_hindrances:
    other_start = _parse_date(other.get("date_of_occurrence"))
    other_end = _parse_date(other.get("date_of_removal")) or today
    if other_start and other_end:
        # Days of overlap between this hindrance and the other
        overlap_start = max(target_start, other_start)
        overlap_end = min(target_end, other_end)
        if overlap_end > overlap_start:
            overlap_days += (overlap_end - overlap_start).days

net_this = max(0, gross_this - overlap_days)
approved = min(net_this, claimed_days) if claimed_days > 0 else net_this
```

### ⚠️ ESCALATION BUG 1 — `days_remaining()` returns 0 for all deadlines instead of correct positive values (MEDIUM — deadline display broken in dashboard)

**EPC NONE → NOTICE_OF_INTENT:**
- Today = `date(2026, 1, 7)`, deadline_days = 60
- Expected deadline = `2026-03-08`, days remaining = 60
- Got: `2026-03-08 (0 days remaining)` ← wrong

**Root cause in `agents/escalation_agent.py` `days_remaining()`:**
```python
def days_remaining(self) -> Optional[int]:
    if not self.tier_deadline:
        return None
    deadline = _parse_date(self.tier_deadline)
    if deadline:
        return max(0, (deadline - date.today()).days)  # ← uses date.today(), NOT the test's fixed_today
    return None
```

`date.today()` returns the real current date (May 2026), not the `fixed_today=Jan 7 2026` passed to `advance_escalation()`. So `(2026-03-08 - 2026-05-01).days = -54` → `max(0, -54) = 0`.

This is only a test environment issue — in production `date.today()` IS correct. But for the deadline display in the dashboard, the countdown is always 0 for past-computed records. It only works correctly for records created today.

**Fix for production correctness — change `days_remaining()` to accept an optional `today` parameter:**
```python
def days_remaining(self, today: date = None) -> Optional[int]:
    today = today or date.today()
    if not self.tier_deadline:
        return None
    deadline = _parse_date(self.tier_deadline)
    if deadline:
        return max(0, (deadline - today).days)
    return None
```

### ❌ EXTRACTION BUG 1 — Milestone trigger_days all return None or wrong (HIGH — without trigger_days, compliance milestone checks cannot fire)

**Root cause:** `extract_milestones()` regex looks for:
```python
r"Day\s+(\d+)\s+from\s+(?:the\s+)?Appointed"
```
The contract DOCX text (after `docx_to_md`) renders the table as:
```
| M-1 | Project Milestone-I: ... | 28% | Day 204 | 20% |
```
The regex expects `"Day 204 from the Appointed Date"` but the table cell just says `"Day 204"`. The phrase `"from the Appointed Date"` doesn't follow in the table cell.

**Fix in `agents/extraction_engine.py` `extract_milestones()`:**
```python
# Add a table-aware pattern as fallback:
day = _int(_find(r"Day\s+(\d+)\s+from\s+(?:the\s+)?Appointed", block))
if day is None:
    # Fallback: look for standalone "Day NNN" in table cell context
    day = _int(_find(r"\|\s*Day\s+(\d+)\s*\|", block))
if day is None:
    # Second fallback: any "Day NNN" in the block
    day = _int(_find(r"\bDay\s+(\d+)\b", block))
```

Similarly for `required_physical_progress_pct` — the table column header is `"Req. Progress"` and the value is `"20%"` but the regex looks for `"(\d+)%\s+physical\s+progress"` which doesn't match table format.

```python
# Add table fallback for progress pct:
progress = _int(_find(r"(\d+)%\s+physical\s+progress", block))
if progress is None:
    progress = _int(_find(r"\|\s*(\d+)%\s*\|", block))  # last | XX% | in block
```

The contract text says: `"a Performance Security equal to 5% of the Contract Price"` — note **"Contract Price"** not "Contract Value". The regex misses "Price".

The contract says `"within 15 days of receipt of the Letter of Acceptance"` — the word `"receipt of"` breaks the match.

### ❌ EXTRACTION BUG 3 — EoT `application_deadline_days` returns None and payment `payment_release_deadline_days` returns None (HIGH)

**EoT root cause:** The contract says `"within 14 (fourteen) days of the occurrence of the event"` — the regex looks for `"within X days of the hindrance"` but the DOCX uses `"within 14 (fourteen) days of the occurrence"`.

**Payment root cause:** The contract says `"The Authority shall release payment within 30 days of certification"` — the regex looks for `"payment shall be released within"` but the actual text is `"Authority shall release payment within"`.

**Fix in `agents/extraction_engine.py`:**
```python

# Payment workflow:
"payment_release_deadline_days": _int(
    _find(r"(?:payment\s+shall\s+be\s+released|shall\s+release\s+payment|release\s+payment)\s+within\s+(\d+)\s+days", text)
```
